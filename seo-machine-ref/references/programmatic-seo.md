# Programmatic SEO Playbooks

## The 12 Playbooks

| Playbook | Pattern | Example |
|----------|---------|---------|
| Templates | "[Type] template" | "resume template" |
| Curation | "best [category]" | "best website builders" |
| Conversions | "[X] to [Y]" | "$10 USD to GBP" |
| Comparisons | "[X] vs [Y]" | "webflow vs wordpress" |
| Examples | "[type] examples" | "landing page examples" |
| Locations | "[service] in [location]" | "dentists in austin" |
| Personas | "[product] for [audience]" | "CRM for real estate" |
| Integrations | "[product A] [product B]" | "slack asana integration" |
| Glossary | "what is [term]" | "what is pSEO" |
| Translations | Content in multiple languages | Localized content |
| Directory | "[category] tools" | "AI copywriting tools" |
| Profiles | "[entity name]" | "stripe ceo" |

## Choosing Your Playbook

| If you have... | Consider... |
|----------------|-------------|
| Proprietary data | Directories, Profiles |
| Product with integrations | Integrations |
| Design/creative product | Templates, Examples |
| Multi-segment audience | Personas |
| Local presence | Locations |
| Tool/utility product | Conversions |
| Content/expertise | Glossary, Curation |
| Competitor landscape | Comparisons |

Multiple playbooks can layer (e.g., "Best coworking spaces in San Diego" = Curation + Locations).

## Data Defensibility Hierarchy

1. Proprietary (you created it) — strongest
2. Product-derived (from your users)
3. User-generated (your community)
4. Licensed (exclusive access)
5. Public (anyone can use) — weakest

## Key Rules

1. **Unique value per page** — Not just swapped variables
2. **Subfolders not subdomains** — `yoursite.com/templates/resume/` not `templates.yoursite.com`
3. **Genuine search intent match** — Pages must actually answer the query
4. **Quality over quantity** — 100 great pages > 10,000 thin ones
5. **No doorway pages** — Must provide real utility

## Implementation Framework

1. **Keyword Pattern Research** — Identify repeating structure, variables, unique combinations, aggregate volume
2. **Data Requirements** — Source data for each page (first-party, scraped, licensed, public)
3. **Template Design** — Header with keyword, unique intro, data-driven sections, related pages, CTAs
4. **Internal Linking** — Hub-spoke model, cross-links between related spokes, no orphan pages
5. **Indexation** — Prioritize high-volume, noindex thin variations, separate sitemaps by type

## Quality Checks

**Pre-launch:**
- [ ] Each page provides unique value
- [ ] Answers search intent
- [ ] Unique titles and meta descriptions
- [ ] Schema markup implemented
- [ ] Connected to site architecture (no orphans)

**Post-launch:**
- Track indexation rate, rankings, traffic, engagement, conversion
- Watch for thin content warnings, ranking drops, manual actions
