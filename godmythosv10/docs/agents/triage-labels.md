# Triage labels

> The 5-state, 2-category state machine every triaged issue moves through.
> Mirrors `references/pocockops-doctrine.md §TRIAGE` from the godmythos skill.

---

## Categories (one per issue)

| Label | Meaning |
|-------|---------|
| `bug` | Something is broken — behavior contradicts intended design or stated documentation |
| `enhancement` | New feature, improvement, or change to intended behavior |

---

## States (one per issue)

| Label | Meaning |
|-------|---------|
| `needs-triage` | Maintainer needs to evaluate. Default on creation. |
| `needs-info` | Waiting on reporter for clarification or repro details. |
| `ready-for-agent` | Fully specified. AFK agent can complete autonomously. |
| `ready-for-human` | Specified, but requires human implementation (judgment, access, design). |
| `wontfix` | Will not be actioned. Closed with rationale. |

---

## State transitions

```
        unlabeled
            ↓
       needs-triage  ←──────────────┐
       │                            │
       ├─→ needs-info ──────────────┘  (reporter replies)
       │
       ├─→ ready-for-agent  ──→ {agent picks up} ──→ {PR opened} ──→ closed
       │
       ├─→ ready-for-human  ──→ {human picks up} ──→ {PR opened} ──→ closed
       │
       └─→ wontfix          ──→ closed
```

A maintainer can override any state at any time.

---

## State role conflicts

If an issue accidentally gets two state roles (e.g., both `needs-triage`
and `ready-for-agent`), **stop and ask the maintainer** before doing
anything else. Do not proceed with assumed precedence.

---

## When agents apply states

| Trigger | Agent action |
|---------|--------------|
| New issue created (no triage label) | Apply `needs-triage` |
| Manual `/triage` command on an issue | Run TRIAGE mode → recommend category + state |
| Reporter replies on a `needs-info` issue | Move back to `needs-triage` for re-evaluation |
| Grilling resolves all open questions | Recommend `ready-for-agent` or `ready-for-human` |
| Issue contradicts an existing ADR | Recommend `wontfix`; reference the ADR |
| Issue resembles an existing `.out-of-scope/` file | Recommend `wontfix`; link the rationale |
| Agent cannot reproduce the bug after honest effort | Recommend `needs-info` with specific questions |

## Triage comment templates

### Recommendation comment (initial triage)

```markdown
> *This was generated by AI during triage.*

## Triage recommendation

**Category:** {bug | enhancement}
**State:** {needs-info | ready-for-agent | ready-for-human | wontfix}

## Reasoning

{1–3 sentences explaining the recommendation.}

## Codebase context relevant to this issue

{Brief summary of relevant modules / ADRs / prior knowledge. Use CONTEXT.md
vocabulary.}

{If reproduction was attempted: state outcome — successful with code path,
failed despite effort, or insufficient detail.}
```

### Needs-info comment

```markdown
> *This was generated by AI during triage.*

## Triage Notes

**What we've established so far:**

- {point}
- {point}

**What we still need from you (@{reporter}):**

- {specific, actionable question}
- {specific, actionable question}
```

### Wontfix (bug)

```markdown
> *This was generated by AI during triage.*

## Closing as wontfix

{Polite explanation of why this won't be addressed. Reference the relevant
ADR or design intent. Suggest workarounds where possible.}

If your use case is meaningfully different from what we considered, please
reopen with the additional context.
```

### Wontfix (enhancement)

```markdown
> *This was generated by AI during triage.*

## Closing as wontfix

{Brief explanation. Full rationale recorded in `.out-of-scope/{slug}.md`.}

Conditions under which we'd reconsider:

{From the .out-of-scope file — what would have to change for this to be
worth revisiting.}
```

---

## Disclaimer requirement

Every comment generated during AI triage **must** start with:

```markdown
> *This was generated by AI during triage.*
```

This is non-negotiable. Maintainers and contributors need to know which
comments came from an agent so they can apply appropriate scrutiny.
